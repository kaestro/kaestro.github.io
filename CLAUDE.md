# kaestro.github.io 프로젝트 가이드

## 블로그 글 작성

블로그 글(`docs/_posts/**`)을 작성·수정할 때는 [.claude/rules/blog-writing-guide.md](.claude/rules/blog-writing-guide.md) 의 가이드라인을 따른다 — Front Matter 필수 항목, 글 구조 설계, 톤·스타일, 줄바꿈/문장 규칙, 이미지 경로, 디렉토리 구조. (해당 룰은 `docs/_posts/**` 작업 시 자동 로드된다.)

특히 잊지 말 것:

- **읽히는 에세이로**: 감정·비유·대화체를 살리고, 건조한 기술 보고서 톤은 피한다
- **흥미 우선 + 개요 먼저**: 가장 흥미로운 내용을 앞으로 끌어오고, 도입 개요로 전체 그림을 먼저 제시한다
- **실제 경험만**: 해보지 않은 경험을 날조하지 않고, 내부 이슈 번호(MF-xxx)는 글에 노출하지 않는다
